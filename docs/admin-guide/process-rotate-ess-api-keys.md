# Rotate ESS API Keys

## Overview

The ESS API keys are used to authenticate the ESS API requests. The ESS API keys are generated by a workflow and are valid for 1 day. The ESS API keys can be rotated manually by executing a [workflow](https://github.com/elastic/observability-test-environments/blob/main/.github/workflows/cluster-rotata-api-keys.yml).
The keys are rotated every day automatically by the workflow.

It is possible to rotated the ESS API keys manually by executing the same commands the workflow executes.

```bash
make -C ansible rotate-ess-api-key-staging
make -C ansible rotate-ess-api-key-qa
make -C ansible rotate-ess-api-key-pro
```

It is possible to update the console Api Keys too.
The production ESS console is not accessible from internet,
so to update the console Api Keys require VPN connection to production environment.
The following commands can be used to update the console and user Api Keys with a VPN connection to production environment.

```bash
make -C ansible rotate-ess-console-api-key-staging
make -C ansible rotate-ess-console-api-key-qa
make -C ansible rotate-ess-console-api-key-pro
```
