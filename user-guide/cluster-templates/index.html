    Cluster templates¶

    Overview¶

    The oblt-cli templates are a way to create many clusters with a similar configuration. The templates are Go template files that are rendered by the oblt-cli tool. Each template is a cluster configuration file using the oblt-cli cluster configuration format, and some Go variable substitutions. The templates are stored in environment/users and have the extension .yml.tmpl. A template must have a name template_name and a description template_description. The description should explain the parameters you need to pass to the template. There are some parameters introduced by the tool automatically, they cannot be overwritten, These parameters are:

    • ClusterName: autogenerated name for the new cluster.
    • Date: timestamp of the operation.
    • SlackChannel: Slack channel configured for the tool to report the secrets.
    • User: the username to tag the deploys.

    To deploy a template we will use the operation oblt-cli cluster create custom something like the following command:

    oblt-cli cluster create custom --template ccs \
  --cluster-name-prefix 'prefix' \
  --parameters '{
    "ElasticsearchDockerImage":"docker.elastic.co/cloud-ci/elasticsearch:8.0.0-alpha2-1b2647e3",
    "RemoteClusterName":"edge-oblt",
    "StackVersion":"8.0.0-alpha1"
    }'

    this command uses the ccs template directly instead the cluster create ccs command.

    To list the available templates you can use the oblt-cli cluster templates command:

    CCS¶

    Cross Cluster Search (CCS) template configures a oblt cluster that has a remote cluster configured for CCS. Parameters:

    • ClusterName: Name for the new oblt cluster (automatic).
    • RemoteClusterName: Oblt cluster name to configure as remote cluster (release-oblt, dev-oblt, edge-oblt, ...).
    • Target: Target environment to deploy (qa, staging, production) (default: production)
    • TrustedOrg: Trusted organization to use for CCS (Optional).
    • StackVersion: Elastic Stack version used for the new cluster.
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • EphemeralCluster: Whether the cluster is ephemeral (Optional)(default false).
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    Usage:

      oblt-cli cluster create ccs \
    --remote-cluster=dev-oblt \
    --cluster-name-prefix mycustomcluster

    It is possible to use the template directly:

      oblt-cli cluster create custom \
      --template ccs \
      --parameters '{
          "RemoteClusterName":"dev-oblt",
          "StackVersion":"8.3.0-SNAPSHOT",
          "Target": "qa",
          "TrustedOrg": "1176847267",
          "ElasticsearchDockerImage":"docker.elastic.co/observability-ci/elasticsearch-cloud:8.3.0-SNAPSHOT"
          }'

      oblt-cli cluster create custom \
      --template ccs \
      --parameters '{
          "RemoteClusterName":"dev-oblt",
          "StackVersion":"8.3.0-SNAPSHOT",
          "ElasticsearchDockerImage":"docker.elastic.co/observability-ci/elasticsearch-cloud:8.3.0-SNAPSHOT",
          "EphemeralCluster": true
          }'

    Deploy Kibana¶

    Deploy Kibana template configure a oblt cluster that has a remote cluster configured for CCS and deploy a custom Kibana Docker image. Parameters:

    • ClusterName: Name for the new oblt cluster (edge-oblt-abpag-ccs, edge-oblt-etagt-ccs)(automatic).
    • RemoteClusterName: Oblt cluster name to configure as remote cluster (release-oblt, dev-oblt, edge-oblt, ...).
    • StackVersion: Elastic Stack version used for the new cluster.
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is a SNAPSHOT (docker.elastic.co/staging/elasticsearch-cloud:8.0.0-beta1-d3622a20, docker.elastic.co/cloud-release/elasticsearch-cloud:8.0.0-SNAPSHOT)(Optional).
    • KibanaDockerImage: Docker image to use for Kibana (docker.elastic.co/kibana/kibana:8.0.0-SNAPSHOT).
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    Usage:

      oblt-cli cluster create custom \
      --template deploy-kibana \
      --parameters '{
          "RemoteClusterName":"dev-oblt",
          "StackVersion":"8.3.0-SNAPSHOT",
          "ElasticsearchDockerImage":"docker.elastic.co/observability-ci/elasticsearch-cloud:8.3.0-SNAPSHOT",
          "KibanaDockerImage":"docker.elastic.co/observability-ci/kibana-cloud:8.3.0-SNAPSHOT
          "}'

      oblt-cli cluster create custom \
      --template-file $(pwd)/environments/users/deploy-kibana.yml.tmpl \
      --parameters '{
          "RemoteClusterName":"dev-oblt",
          "StackVersion":"8.3.0-SNAPSHOT",
          "ElasticsearchDockerImage":"docker.elastic.co/observability-ci/elasticsearch-cloud:8.3.0-SNAPSHOT",
          "KibanaDockerImage":"docker.elastic.co/observability-ci/kibana-cloud:8.3.0-SNAPSHOT"
          }'

    deploy-kibana

    ESS¶

    Elastic Cloud (ESS) template configure a oblt cluster that runs on ESS dev region on production. Parameters:

    • ClusterName: Name for the new oblt cluster (automatic).
    • StackVersion: Elastic Stack version used for the new cluster.
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ProductType: Type of deployment you want to make elasticsearch, observability, or security (default observability).
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • KibanaDockerImage: Docker image to use for Kibana. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • ElasticAgentDockerImage: Docker image to use for Elastic Agent. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).
    • ZonesIntegrations: Zones to deploy the integrations server (Optional).
    • ZonesKibana: Zones to deploy the Kibana server (Optional).
    • ZonesElasticsearch: Zones to deploy the Elasticsearch server (Optional).
    • ElasticsearchMemory: Memory for the Elasticsearch server (Optional).
    • KibanaMemory: Memory for the Kibana server (Optional).
    • IntegrationsMemory: Memory for the Integrations server (Optional).

    Usage:

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0",
        "ProductType": "elasticsearch"
        }'

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0",
        "ElasticsearchDockerImage": "docker.elastic.co/observability-ci/elasticsearch-cloud:8.15.0-edd09abf"
        }'

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0",
        "KibanaDockerImage": "docker.elastic.co/observability-ci/kibana-cloud:8.15.0-edd09abf"
        }'

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0",
        "ElasticAgentDockerImage": "docker.elastic.co/observability-ci/elastic-agent-cloud:8.15.0-edd09abf"
        }'

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0-SNAPSHOT",
        "ZonesIntegrations": "3",
        "ZonesKibana": "3",
        "ZonesElasticsearch": "3"
        }'

      oblt-cli cluster create custom \
    --template ess \
    --parameters '{
        "StackVersion": "8.15.0-SNAPSHOT",
        "ElasticsearchMemory": "16",
        "KibanaMemory": "8",
        "IntegrationsMemory": "4"
        }'

    ess

    Serverless¶

    • UpdateMode: Update mode for the cluster (default: recreate)
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    APM Soak¶

    Parameters:

    • ClusterName: Name for the new oblt cluster (automatic).
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ProjectType: Type of project to deploy (elasticsearch, observability, security) (default: observability)
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • KibanaDockerImage: Docker image to use for Kibana. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • FleetDockerImage: Docker image to use for Elastic Agent. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • Target: Target environment to deploy (qa, staging, production) (default: qa)
    • EphemeralCluster: Whether the cluster is ephemeral (Optional)(default false).
    • ApmScenario: APM scenario to use for the cluster (Optional)(default qa).
    • ApmPerfImage: APM Perf docker image (Optional)(default docker.elastic.co/observability-ci/apm-perf:latest-1695914783-e98c86d).
    • MKICluster: MKI cluster to use for the cluster (Optional).
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    Usage:

        oblt-cli cluster create custom \
      --template apm-soak

      --parameters '{"ApmScenario": "qa"}'

        oblt-cli cluster create custom \

        oblt-cli cluster create custom \

    Serverless MIS¶

    Serverless template configures an observability cluster that runs on Elastic Stack serverless on your target environment. Parameters:

    • ClusterName: Name for the new oblt cluster (automatic).
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ProjectType: Type of project to deploy (elasticsearch, observability, security) (default: observability)
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • KibanaDockerImage: Docker image to use for Kibana. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • FleettDockerImage: Docker image to use for Elastic Agent. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • Target: Target environment to deploy (qa, staging, production) (default: qa)
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    The only difference to the serverless template is that apps are not installed.

    Serverless Load test¶

    Serverless Load test template configure a oblt cluster that runs on Elastic Stack serverless on the production environment. Then launches a load test using opbeans-go. Parameters:

    • ClusterName: Name for the new oblt cluster (automatic).
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • KibanaDockerImage: Docker image to use for Kibana. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • FleettDockerImage: Docker image to use for Elastic Agent. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • Target: Target environment to deploy (qa, staging, production) (default: qa)
    • NumberOfOpbeansReplicas: Number of opbeans replicas to deploy (default: 10)
    • BatchLength: Time in seconds to run each batch (default: 120)
    • RequestPerMinute: Number of requests per minute to run (default: 4000)
    • Parallelism: Number of Opbeans-loadgen pods to run in parallel (default: 10)
    • GitOps: Whether GitOps is enabled (Optional).
    • GitHubCommit: For GitOps, the Git commit that requested this cluster (Optional).
    • GitHubIssue: For GitOps, the GitHub issue that requested this cluster (Optional).
    • GitHubPullRequest: For GitOps, the GitHub Pull Request that requested this cluster (Optional).
    • GitHubRepository: For GitOps, the GitHub Repository, ORG/REPO, that requested this cluster (Optional).
    • GitHubCommentId: For GitOps, the GitHub comment Id that requested this cluster (Optional).

    The only difference to the serverless template is that apps are not installed.

    Usage:

      oblt-cli cluster create custom \
    --template serverless-load-test

      oblt-cli cluster create custom \
    --template serverless-load-test \
    --parameters '{"Target": "qa"}'

      oblt-cli cluster create custom \
    --template serverless-load-test \
    --parameters '{
        "NumberOfOpbeansReplicas": 100,
        "BatchLength": 60,
        "RequestPerMinute": 4000,
        "Parallelism": 100
        }'

    Stack Profiling¶

    Deploy The Elastic Stack on k8s with ECK, all Opbeans, and configure Stack monitoring and profiling to send data to monitoring-oblt. It is possible to turn off some components of the Stack and set custom Docker images for each component. This template is designed to monitor the performance of the Elastic Stack.

    Note

    CCS is not supported in this template. To replace the CCS functionality, we deploy the Opbeans to generated data.

    Parameters:

    • ClusterName: Name for the new oblt cluster (edge-oblt-abpag-ccs, edge-oblt-etagt-ccs)(automatic).
    • StackVersion: Elastic Stack version used for the new cluster.
    • SlackChannel: SlackChannel name or Slack member ID to send the credentials of the cluster (#my-channel, @MIDNUM)(automatic)
    • ElasticsearchEnabled: Enable Elasticsearch (true, false)(Optional).
    • ElasticsearchDockerImage: Docker image to use for Elasticsearch. This version should be the same build of the remote cluster if the version is a SNAPSHOT (docker.elastic.co/staging/elasticsearch-cloud:8.0.0-beta1-d3622a20, docker.elastic.co/cloud-release/elasticsearch-cloud:8.0.0-SNAPSHOT)(Optional).
    • KibanaEnabled: Enable Kibana (true, false)(Optional).
    • KibanaDockerImage: Docker image to use for Kibana (docker.elastic.co/kibana/kibana:8.0.0-SNAPSHOT).
    • ElasticAgentEnabled: Enable Elastic Agent (true, false)(Optional).
    • ElasticAgentDockerImage: Docker image to use for Elastic Agent. This version should be the same build of the remote cluster if the version is an SNAPSHOT(Optional).
    • OpbeansEnabled: Enable Opbeans (true, false)(Optional).

    Usage:

      oblt-cli cluster create custom \
      --template stack-profiling \
      --parameters '{"StackVersion":"8.7.0"}'

      oblt-cli cluster create custom \
      --template stack-profiling \
      --parameters '{
        "StackVersion":"8.7.0-SNAPSHOT",
        "ElasticsearchEnabled": true,
        "KibanaEnabled": false,
        "ElasticAgentEnabled": false,
        }'

      oblt-cli cluster create custom \
      --template stack-profiling \
      --parameters '{
        "StackVersion":"8.7.0-SNAPSHOT",
        "ElasticsearchEnabled": true,
        "ElasticsearchDockerImage":"docker.elastic.co/observability-ci/elasticsearch-cloud:8.7.0-SNAPSHOT",
        "KibanaEnabled": true,
        "KibanaDockerImage":"docker.elastic.co/observability-ci/kibana-cloud:8.7.0-SNAPSHOT",
        "ElasticAgentEnabled": true,
        "ElasticAgentDockerImage":"docker.elastic.co/observability-ci/elastic-agent-cloud:8.7.0-SNAPSHOT"
        }'
